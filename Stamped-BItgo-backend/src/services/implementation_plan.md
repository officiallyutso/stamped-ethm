# User Auto-Wallet & Earnings UI — Implementation Plan

## Summary

Two changes to the existing workspace payout system:

1. **Auto-generate payout wallet for users** — when a user joins a workspace, the backend creates a BitGo wallet for them automatically. No manual address input. The workspace owner sends money to this auto-generated address.

2. **My Earnings + Checkout UI** — show user's earnings in Settings, with a "Checkout" button that collects an external address for withdrawal (UI-only for now).

---

## 1. Firestore Schema Change

### New Collection: `workspace_members/{docId}`

Each document represents a user's membership in a workspace, including their auto-generated wallet.

```json
{
  "workspaceId": "ws_abc",
  "userId": "uid_def456",
  "walletId": "bitgo_wallet_id",
  "walletAddress": "0xAutoGenerated...",
  "walletBalance": "0.008",
  "walletBalanceWei": "8000000000000000",
  "joinedAt": "<timestamp>"
}
```

> [!NOTE]
> We use a separate collection instead of storing on `users/{uid}` because a user can be in multiple workspaces, each with a different wallet.

## 2. Backend Changes

### [MODIFY] [workspaceWalletService.ts](file:///d:/projects/STAMPED/stamped-app/stamped/Stamped-BItgo-backend/src/services/workspaceWalletService.ts)

Add two new methods:

#### `createUserWallet(workspaceId, userId, displayName)` 
1. Check if `workspace_members` doc already exists for this user+workspace → return existing if so
2. Call `bitgo.coin().wallets().generateWallet(...)` with label `"User Wallet - {displayName} - {workspaceId}"`
3. Store `walletId`, `walletAddress` in `workspace_members/{autoId}` doc
4. Return wallet info

#### `getUserWalletInfo(workspaceId, userId)`
1. Query `workspace_members` where `workspaceId` + `userId`
2. Return wallet address, balance info
3. Optionally refresh balance from BitGo
