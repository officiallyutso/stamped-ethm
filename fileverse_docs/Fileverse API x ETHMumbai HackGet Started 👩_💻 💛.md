What you get 🍱 

LLM-optimized

dDocs provides an llm.txt route endpoint that gives LLMs context on how to interact with and use dDocs. This helps AI agents understand the available capabilities and API structure automatically.

ClawHub Skill 

The API includes an agent skill available on ClawHub, that you can plug it directly into your agents. 

End-to-End Encryption

Documents are encrypted on your device before they are published onchain. Only you hold the decryption keys, not even Fileverse can read your documents.

Developer Space

Your sealed, encrypted environment for programatically accessible document in your dDocs homepage (find it in “Default API space” after enabling developer mode in settings). 

It operates with separate encryption keys: your personal documents remain completely separate, external tools connected via API do not have access to them. 

Fileverse API Server

The local or cloud-hosted server that handles API requests and document management. 

Onchain Verification

Documents are published onchain for permanent, immutable storage with cryptographic proof of creation and modifications.

API-First Design

No built-in UI: interact with dDocs entirely through the REST API, CLI tools, or integrations.
